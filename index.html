文件系统设计与实现【6】 - 本地快速文件系统（FFS）

　　最早的文件系统是随着 Unix 操作系统发布第一个版本时发布，他是由 Unix 之父 Ken Thompson 研发，我们现在称他为“旧版 Unix 文件系统”，这个文件系统非常的简单，在硬盘上的数据结构如下图所示：


　　超级块（S）包含了整个文件系统的信息：磁盘卷的大小，inode 的数量，指向空闲块列表头的指针等等。磁盘的 inode 区域包含了整个文件系统所有的 inode 节点，剩下的大部分磁盘区域都属于数据区。
　　这个旧版 Unix 文件系统的优点就是非常的简单，并且已经支持了文件系统对数据的抽象化 - 文件和目录结构的分层。这个简单易用的文件系统相对于过去既笨拙又只提供一层目录结构的早期存储系统来说是一个真正的进步。

文件系统设计与实现【5】 - 简单文件系统实现

　　这一章开始，我们将慢慢深入对文件系统的理解。本章将介绍一个简单的文件系统 vsfs（very simple file system）。他是一个典型的 UNIX 文件系统的简化版本，被我们用来介绍一些基础的，但也在目前文件系统中大量运用的磁盘结构、访问方法和策略等。
　　文件系统是一个纯粹的软件，不像我们开发 CPU 和虚拟内存一样，需要添加一些硬件特性的支持。由于建立文件系统具有极大的灵活性，市面上存在了包括 AFS 到 ZFS 各种类型的文件系统，他们各自具有不同的数据结构，都发挥着各自的优势，自然，也会有自己的劣势。因此我们学习文件系统的方法就是通过实例进行学习：首先这一章介绍一个简单的文件系统 vsfs，然后继续学习一系列真实的文件系统，理解他们在实践中的不同。

文件系统设计与实现【4】 - 文件与目录

　　本书前文中我们已经了解到了操作系统两个关键的抽象概念：一个是进程，他是 CPU 的虚拟化；一个是地址空间，他是内存的虚拟化。这两者的结合，允许程序运行在一个独立的、私有的空间中，就好像他拥有自己独立的处理器和内存。这种虚拟化让系统编程变得更简单。现在不但桌面和服务器上普遍在使用这种虚拟化，同时也在各种可编程平台上快速发展，比如移动设备。
　　在后续部分，我们介绍一个更为关键的虚拟化部分：持久性存储。一个进行持久存储的设备（传统磁盘或者 SSD），不同有内存，掉电后会失去数据，他能安全的、永久的保存数据（至少能够长时间保存数据）。因此，操作系统必须额外关注这种设备，因为他可以用来保存用户重要的数据信息。
　　后续的章节中，我们将开始研究这种持久性数据的管理方式，关注能够提高系统性能和可靠性的方法。这一章介绍与 UNIX 文件系统进行交互的接口。
